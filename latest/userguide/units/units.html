Units & Quantities¶

Spectrochempy can do calculations with units - it uses pint to define and perform operation on data with units.

[1]:

from spectrochempy import *  # noqa: F403
  SpectroChemPy's API - v.0.2.5.dev8
© Copyright 2014-2021 - A.Travert & C.Fernandez @ LCS

Create quantities¶

to create quantity, use for instance, one of the following expression:

[2]:

Quantity('10.0 cm^-1')

[2]:
10.0 cm-1
[3]:

Quantity(1.0, 'cm^-1/hour')

[3]:
1.0 cm-1.hr-1
[4]:

Quantity(10.0, ur.cm / ur.km)

[4]:
10.0 scaled-dimensionless (1e-05)

or may be (?) simpler,

[5]:

10.0 * ur.meter / ur.gram / ur.volt

[5]:
10.0 m.V-1.g-1

ur stands for unit registry, which handle many type of units (and conversion between them)

Do arithmetics with units¶

[6]:

a = 900 * ur.km
b = 4.5 * ur.hours
a / b

[6]:
200.0 km.hr-1

Such calculations can also be done using the following syntax, using a string expression

[7]:

Quantity("900 km / (8 hours)")

[7]:
112.5 km.hr-1

Convert between units¶

[8]:

c = a / b
c.to('cm/s')

[8]:
5555.555555555556 cm.s-1

We can make the conversion inplace using ito instead of to

[9]:

c.ito('m/s')
c

[9]:
55.55555555555556 m.s-1

Do math operations with consistent units¶

[10]:

x = 10 * ur.radians
np.sin(x)

[10]:
-0.5440211108893698

Consistency of the units are checked!

[11]:

x = 10 * ur.meters
np.sqrt(x)

[11]:
3.1622776601683795 m0.5

but this is wrong…

[12]:

x = 10 * ur.meters
try:
    np.cos(x)
except DimensionalityError as e:
    error_(e)

 ERROR | DimensionalityError: Cannot convert from 'meter' ([length]) to 'radian' (dimensionless)
